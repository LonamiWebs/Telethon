SendReactionRequest

Both users and bots may be able to use this request. See code examples.

---functions---
messages.sendReaction#d30d78d4 flags:# big:flags.1?true add_to_recent:flags.2?true peer:InputPeer msg_id:int reaction:flags.0?Vector<Reaction> = Updates

Returns

Updates

This type can be an instance of either:

UpdateShortUpdateShortChatMessage
UpdateShortMessageUpdateShortSentMessage
UpdatesUpdatesCombined
UpdatesTooLong

Parameters

peerInputPeerAnything entity-like will work if the library can find its Input version (e.g., usernames, Peer, User or Channel objects, etc.).
msg_idint
bigflagThis argument defaults to None and can be omitted.
add_to_recentflagThis argument defaults to None and can be omitted.
reactionReactionThis argument defaults to None and can be omitted. Otherwise, a list must be supplied.

Known RPC errors

This request can't cause any RPC error as far as we know.

Example

from telethon.sync import TelegramClient
from telethon import functions, types

with TelegramClient(name, api_id, api_hash) as client:
    result = client(functions.messages.SendReactionRequest(
        peer='username',
        msg_id=42,
        big=True,
        add_to_recent=True,
        reaction=[types.ReactionEmoji(
            emoticon='some string here'
        )]
    ))
    print(result.stringify())
